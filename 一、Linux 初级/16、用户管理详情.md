2、MD5校验

```bash
md5sum filename       #生成校验码+文件名
md5sum -c file.log    #将文件中存放的文件名和校验码取出（不一定是log文件）
							a.执行md5sum filename 生成新校验码
							b.比对校验码


注意：文件名要使用绝对路径，否则校验文件移动后，还是检验相对路径，就会失效。


1.作用
	MD5 就是文件唯一身份指纹，预设好标准指纹值
	二次校验比对不一致，就代表文件损坏、数据丢失或是被人为篡改

2.使用方法
	基础流程：
		（1）生成md5值
			[root@breadbomb ~]# cat a.txt
			hello world
			[root@breadbomb ~]# md5sum a.txt
			6f5902ac237024bdd0c176cb93063dc4  a.txt
		
		（2）修改文件内容
			[root@breadbomb ~]# echo world hello > a.txt
			[root@breadbomb ~]# cat a.txt
			world hello
		
		（3）重新校验，校验值发生变化
			[root@breadbomb ~]# md5sum a.txt
			baade5592c37e1ecbac490c16ef5671c  a.txt
	
	进阶流程：
		（1）生成md5值，放入文件中
			[root@breadbomb ~]# cat a.txt
			hello world
			[root@breadbomb ~]# md5sum a.txt > md5.log
		
		（2）检验md5.log （检验结果没变）
			[root@breadbomb ~]# md5sum -c md5.log
			a.txt: OK
		
		（3）检验md5.log （检验结果变化）
			[root@breadbomb ~]# echo world hello > a.txt
			[root@breadbomb ~]# md5sum -c md5.log
			a.txt: FAILED
			md5sum: WARNING: 1 computed checksum did NOT match
		
		（4）结合定时任务和脚本实现实时监控
			每5分钟检测此文件输出结果，如果出现FAILD则发送邮件告警通知给相关人员
		
		检验流程：
			（1）md5sum a.txt 会生成一行校验码+文件名
			（2）将校验码+文件名存入日志文件
			（3）md5 -c 检验日志文件时
				a.先获取文件名，执行md5sum a.txt，生成新校验码
				b.将新校验码与日志文件中校验码比对
					相同： 文件名：OK
					不同： 文件名：FAILED
	
	多文件校验：
		（1）md5sum /root/*.txt > /root/a.log
		（2）md5sum -c /root/a.log
	
	递归文件校验：
		（1）find /root/dir -type f|xargs md5sum > /root/a.log
		（2）md5sum -c /root/a.log
```
