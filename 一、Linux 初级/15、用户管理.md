
1、操作系统开机启动流程

```bash
1.开机BIOS自检
2.MBR引导
3.grub引导菜单
4.加载内核kernel
5.启动init进程
6.读取inittab文件，执行rc.sysinit.rc等脚本
7.启动mingetty,进入系统登录界面

	开机BIOS自检（出现故障频率排序）：
		硬盘、内存、电源、主板、CPU
		
		1.硬盘出现问题
			（1）.是否在质保期内 3年
				a.质保期内，联系售后换新
				b.联系IDC机房，定时间，带身份证，带售后去换
			（2）.过了质保期
				是否有库存，没有直接采购，询问领导更换硬盘流程
		
		2.内存 内存不会坏
			（1）兼容性问题 关机重启 或 刚开机无法启动服务器
				a.重新插拔内存，擦拭金手指
					一根一根往下拔 拔一根启动一下
					全部拔下一根一根插 插一下启动一下
		
		3.电源 坏了直接换 服务器支持两块电源
		4.主板 坏了直接换
		5.cpu 风扇坏，监控cpu温度



1.让一个服务开机自动运行的方法
	（1）被systemctl所管理的服务，直接设置enable
		systemctl enable NetworkManager #开机自动运行NetworkManager服务
	
	（2）不被systemctl所管理的服务，写入/etc/rc.local
		vim /etc/rc.local
	
	（3）写入/etc/profile 环境变量 不建议
```

2、操作系统用户分类

```bash
用户类型：
	（1）root    管理员        拥有最高权限
	（2）name    普通用户      拥有个人用户下的最高权限（优化系统，安全）
	（3）虚拟用户 傀儡用户      启动进程必须使用一个用户来运行，此用户不需要登录操作系统，只为了运行程序存在

系统分类表示方式： 使用UID（系统默认，可修改） 可以在/etc/passwd中第3列查看
	（1）管理员     0
	（2）普通用户   1000+
	（3）虚拟用户   1-999
```
