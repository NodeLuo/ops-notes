
1、操作系统开机启动流程

```bash
1.开机BIOS自检
2.MBR引导
3.grub引导菜单
4.加载内核kernel
5.启动init进程
6.读取inittab文件，执行rc.sysinit.rc等脚本
7.启动mingetty,进入系统登录界面

	开机BIOS自检（出现故障频率排序）：
		硬盘、内存、电源、主板、CPU
		
		1.硬盘出现问题
			（1）.是否在质保期内 3年
				a.质保期内，联系售后换新
				b.联系IDC机房，定时间，带身份证，带售后去换
			（2）.过了质保期
				是否有库存，没有直接采购，询问领导更换硬盘流程
		
		2.内存 内存不会坏
			（1）兼容性问题 关机重启 或 刚开机无法启动服务器
				a.重新插拔内存，擦拭金手指
					一根一根往下拔 拔一根启动一下
					全部拔下一根一根插 插一下启动一下
		
		3.电源 坏了直接换 服务器支持两块电源
		4.主板 坏了直接换
		5.cpu 风扇坏，监控cpu温度



1.让一个服务开机自动运行的方法
	（1）被systemctl所管理的服务，直接设置enable
		systemctl enable NetworkManager #开机自动运行NetworkManager服务
	
	（2）不被systemctl所管理的服务，写入/etc/rc.local
		vim /etc/rc.local
	
	（3）写入/etc/profile 环境变量 不建议
```

2、操作系统用户分类

```bash
用户类型：
	（1）root    管理员        拥有最高权限
	（2）name    普通用户      拥有个人用户下的最高权限（优化系统，安全）
	（3）虚拟用户 傀儡用户      启动进程必须使用一个用户来运行，此用户不需要登录操作系统，只为了运行程序存在

系统分类表示方式： 使用UID（系统默认，可修改） 可以在/etc/passwd中第3列查看
	（1）管理员     0
	（2）普通用户   1000+    普通用户只在自己家目录有开放权限，/tmp里面只能处理自己的东西
	（3）虚拟用户   1-999
```

3、用户相关文件

```bash
1./etc/passwd

[root@breadbomb ~]# cat /etc/passwd
root:x:0:0:root:/root:/bin/bash

第一列：root 用户名称 管理员、普通用户和虚拟用户体现的位置
第二列：x     用户密码
第三列：0     用户UID
第四列：0     用户的GID 用户默认组的ID
第五列：root  描述信息，可写可不写
第六列：/root 用户的家目录
第七列：/bin/bash 命令解释器
		（1）/bin/bash 普通用户，允许登录操作系统
		（2）/sbin/nologin 虚拟用户，不允许登录操作系统

切换用户 ： su - username  普通用户可以登录，虚拟用户不让登录
退出登录 ： exit

2./etc/shadow 用户密码信息

3./etc/group 用户组信息

4./etc/gshadow 用户组密码信息

5./etc/skel 所有用户的家模板
	创建用户流程：
		（1）创建用户 useradd name
		（2）系统将 /etc/skel/隐藏的环境变量文件 复制到 /home/name/家目录中隐藏文件
		（3）系统将属主属组修改为普通用户


问题：出现了bash版本号怎么办?
	具体情况示例：
		[root@breadbomb ~]# 
		变成
		-bash-4.2#
	
	PS1变量：修改[root@breadbom ~]的展示效果的，可以显示不同信息或颜色
	
	（1）注释掉/etc/profile中的PS1变量（若没有就不管）
	（2）删除隐藏文件重新登录系统
		a. rm -rf .bash_profile .bashrc
		b. exit
	（3）解决办法
		cp /etc/skel .* . #将/etc/skel中所有文件复制到家目录中
		ls -a .bash*      #查看是否复制成功
	（4）退出重新即可恢复（执行完第3步）
	（5）不退出可以使用. 或 source 让文件执行一次
		. .bash_profile 或 source .bash_profile
	
	修改PS1变量方法：
		a.临时修改PS1变量,PS1='[\u \t@\h \w]\$'
		b.永久修改写入/etc/profile

```

4、用户管理相关命令

```bash
1.useradd 添加用户
	（1）语法格式
		useradd [参数选项] name
	（2）参数
		-s 指定解释器 /bin/bash /sbin/nologin 创建普通用户或虚拟用户
		-u 指定UID,若不指定则默认往后排
		-g 指定GID,若不指定则默认往后排，指定组必须存在
		-M 不创建家目录
		-G 附加组
	
	示例：
		#创建虚拟用户 指定uid 不建家
		[root@breadbomb ~]# useradd -u 1002 -M -s /sbin/nologin bob
		
		#创建虚拟用户 指定uid 指定gid


2.id 查看用户
	（1）语法格式
		id name
	
	示例：
		[root@breadbomb ~]# id bob
		uid=1002(bob) gid=1002(bob) groups=1002(bob)

3.groupadd 创建组
	（1）语法格式
		groupadd [参数选项] 组名
	（2）参数
		-g GID  指定组ID
		-r      创建系统组，GID通常1-999，用于服务/进程账号
		-f      组已存在时不报错，直接成功退出
		-o      允许创建重复GID的组
		-p      为组设置密码
	
	示例：
		#没有组需要先创建组再创建用户
		[root@breadbomb ~]# groupadd -g 888 bobhome
		[root@breadbomb ~]# useradd -u 1002 -g 888 -M -s /sbin/nologin bob
		[root@breadbomb ~]# id bob
		uid=1002(bob) gid=888(bobhome) groups=888(bobhome)

4.userdel 删除用户
	（1）语法格式
		userdel [参数选项] name
	（2）参数
		-r 删除用户所有相关文件
		-f 强制删除
	
	注意：若不用-r删除，再次创建同名用户时，可能会由于遗留文件引发权限问题
	
	示例：
		userdel -r bob

5.groupdel 删除组
	（1）语法格式
		groupdel name

6.passwd 给用户设置密码
	（1）语法格式
		passwd  #直接回车，修改自己密码
		passwd [参数选项] name
	（2）参数
		-l 锁定用户密码，禁止登录
		-u 解锁用户密码
		-d 删除用户密码（无密码登录）
		-e 强制用户下次登录改密码
		-s 查看用户密码状态
	
	示例：
		[root@breadbomb ~]# passwd
		Changing password for user root.
		New password: 
		BAD PASSWORD: The password is a palindrome
		Retype new password: 
		passwd: all authentication tokens updated successfully.
		
		#给普通用户设置密码后，可以通过xshell连接虚拟机
		#管理员su过去不用输密码，普通用户su到root要密码
		#xshell直接复制粘贴会话，修改用户名、密码，可以直接连接节省步骤
		
		#免jiao

```
