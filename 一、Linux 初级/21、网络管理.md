
1、路由器、交换机

```bash
1.交换机(二层、三层)
	a.作用：实现同一局域网内主机通信，基于MAC地址转发数据
	b.特点
		（1）无法隔离广播域，网络规模过大产生网络风暴
		（2）二层交换机：数据转发，数据链路层
		（3）三层交换机：集成路由功能，可划分VLAN、跨网段通信
	c.通信原理
		数据携带 源MAC地址 + 目的MAC地址，初期通过广播寻址，维护MAC地址表

2.路由器
	a.作用：实现不同网段/局域网之间的通信，基于IP地址转发数据包，天然隔离广播域
	b.特点
		（1）每个接口对应一个独立广播域、独立网段
		（2）依靠路由表选择数据转发路径、遵循最短路径优先
		（3）跨网段通信必须配置网关，路由器接口IP === 网段网关
	c.身份标识
		接口使用IP地址
```

2、外网访问流程：

```bash
主机携带源地址和端口+目标地址经过路由器进行NAT转换成公网IP和端口，找到目标IP后，服务器回应消息，发送源IP+目标IP和端口，再次经过NAT转换，回到主机对应端口

所有端口：25565

一台主机大概有：50-70个端口

一个IP大概有：25565/50-70  个主机
```

## 二、路由器基础 IP 配置（Cisco 设备命令）

### 1. 设备模式切换（前置必学）


```
Router>                // 用户模式（仅查看基础信息）
Router>enable          // 进入特权模式
Router#                // 特权模式（查看路由表、测试ping）
Router#configure terminal  // 进入全局配置模式
Router(config)#        // 全局配置模式（全局参数、命名、路由配置）
Router(config)#exit    // 逐级退出模式
Router(config-if)#exit // 接口模式退回全局模式
```

### 2. 单接口 IP 配置（最常用）

给路由器物理接口配置 IP、启用接口（接口默认关闭）


```
// 1. 进入快速以太网接口fa0/0
Router(config)#interface FastEthernet0/0  
// 简写：int fa0/0

// 2. 配置IP地址 + 子网掩码
Router(config-if)#ip address 192.168.16.1 255.255.255.0  

// 3. 开启接口（关键！接口默认shutdown，必须手动启用）
Router(config-if)#no shutdown  

// 4. 退出接口模式
Router(config-if)#exit
```

### 3. 多接口 IP 配置（多网段互联场景）

一台路由器多个接口，分别对接不同网段，每个接口 IP 属于不同网段：


```
// 配置fa0/0 接口（网段1：192.168.16.0/24）
Router(config)#int fa0/0
Router(config-if)#ip add 192.168.16.1 255.255.255.0
Router(config-if)#no shut
Router(config-if)#exit

// 配置fa0/1 接口（网段2：11.22.33.0/24）
Router(config)#int fa0/1
Router(config-if)#ip add 11.22.33.1 255.255.255.0
Router(config-if)#no shut
Router(config-if)#exit

// 查看直连路由（验证接口配置生效）
Router#show ip route
// 标识 C = connected（直连网段，路由器自身接口网段）
```

### 4. 路由器重命名（区分多台设备）


```
Router(config)#hostname R1   // 将路由器命名为 R1
R1(config)#                  // 命名生效
```

## 三、静态路由配置

### 1. 静态路由原理

手动指定**去往目标网段**的转发路径，适合小型网络、路由路径固定场景；格式统一：

`ip route 目标网段 子网掩码 下一跳IP`

- 目标网段：需要访问的远端网络；
- 子网掩码：目标网段对应的掩码；
- 下一跳 IP：**相邻路由器直连接口 IP**（数据转发的下一站）。

### 2. 拓扑说明（三台路由器串联）

> 拓扑：`R1 <---> R2 <---> R3`
> 
> 网段规划：
> 
> R1：192.168.16.0/24 、互联网段 192.168.2.0/24
> 
> R2：互联网段 192.168.2.0/24 、192.168.3.0/24
> 
> R3：互联网段 192.168.3.0/24 、远端网段 11.22.33.0/24

#### 步骤 1：所有路由器接口 IP 配置

##### R1 配置

ios

```
R1(config)#int fa0/0
R1(config-if)#ip add 192.168.16.1 255.255.255.0
R1(config-if)#no shut
R1(config-if)#exit

R1(config)#int fa0/1
R1(config-if)#ip add 192.168.2.1 255.255.255.0
R1(config-if)#no shut
R1(config-if)#exit
```

##### R2 配置

ios

```
R2(config)#int fa0/0
R2(config-if)#ip add 192.168.2.2 255.255.255.0
R2(config-if)#no shut
R2(config-if)#exit

R2(config)#int fa0/1
R2(config-if)#ip add 192.168.3.1 255.255.255.0
R2(config-if)#no shut
R2(config-if)#exit
```

##### R3 配置

ios

```
R3(config)#int fa0/0
R3(config-if)#ip add 192.168.3.2 255.255.255.0
R3(config-if)#no shut
R3(config-if)#exit

R3(config)#int fa0/1
R3(config-if)#ip add 11.22.33.1 255.255.255.0
R3(config-if)#no shut
R3(config-if)#exit
```

#### 步骤 2：逐台配置静态路由（双向路由，保证往返通信）

##### 1）R1 配置：去往 11.22.33.0/24 网段

ios

```
R1(config)#ip route 11.22.33.0 255.255.255.0 192.168.2.2
// 解释：目标网段11.22.33.0，下一跳为R2接口IP 192.168.2.2

// 查看路由表（S = static 静态路由）
R1#show ip route
```

##### 2）R2 配置：双向路由（去往 R1 网段 + 去往 R3 网段）

ios

```
// 去往 R1 所在网段 192.168.16.0/24
R2(config)#ip route 192.168.16.0 255.255.255.0 192.168.2.1

// 去往 R3 所在网段 11.22.33.0/24
R2(config)#ip route 11.22.33.0 255.255.255.0 192.168.3.2
```

##### 3）R3 配置：回指路由（去往 R1 网段，数据返程）

ios

```
R3(config)#ip route 192.168.16.0 255.255.255.0 192.168.3.1
```

### 3. 静态路由总结

1. 必须**双向配置**：A 到 B、B 到 A 都需要路由，否则只能单通；
2. 适合网络规模小、拓扑固定的场景；
3. 缺点：网络扩容后配置量大、故障无法自动切换路径。

## 四、动态路由 RIP 配置（补充原文缺失内容）

### 1. RIP 基础介绍

1. **全称**：路由信息协议，**距离矢量型动态路由协议**；
2. **适用场景**：中小型局域网、园区网；
3. **版本区分**
    
    - RIP v1：不支持子网划分、不发送掩码，仅广播更新；
    - RIP v2（推荐）：支持子网、无类路由，支持广播 / 组播更新，主流使用；
    
4. **规则**：以**跳数**作为路径判断标准，最大有效跳数 15 跳（16 跳判定为不可达）；
5. **原理**：路由器之间自动交换路由表，无需手动配置路由。

### 2. RIP 通用配置命令（Cisco）

#### 基础开启 RIP 进程

ios

```
// 进入全局模式，开启RIP
Router(config)#router rip  

// 切换为RIP v2（必配，推荐版本）
Router(config-router)#version 2  

// 关闭路由自动汇总（RIP v2常用，保证子网正常通信）
Router(config-router)#no auto-summary  

// 宣告直连网段（仅宣告自身**直连网段**，不带子网掩码）
// 格式：network 主网络号
Router(config-router)#network 网段地址
```

### 3. 基于上面三台路由器拓扑，配置 RIP v2

拓扑不变：`R1 <--> R2 <--> R3`，网段同上。

#### 1）R1 完整 RIP 配置

ios

```
R1(config)#router rip
R1(config-router)#version 2          // 使用RIP2
R1(config-router)#no auto-summary     // 关闭自动汇总
// 宣告自身两个直连网段
R1(config-router)#network 192.168.16.0
R1(config-router)#network 192.168.2.0
R1(config-router)#exit
```

#### 2）R2 完整 RIP 配置

ios

```
R2(config)#router rip
R2(config-router)#version 2
R2(config-router)#no auto-summary
// 宣告自身两个直连网段
R2(config-router)#network 192.168.2.0
R2(config-router)#network 192.168.3.0
R2(config-router)#exit
```

#### 3）R3 完整 RIP 配置

ios

```
R3(config)#router rip
R3(config-router)#version 2
R3(config-router)#no auto-summary
// 宣告自身两个直连网段
R3(config-router)#network 192.168.3.0
R3(config-router)#network 11.22.33.0
R3(config-router)#exit
```

### 4. RIP 验证命令（排查 & 查看）

ios

```
// 1. 查看路由表，学习到的RIP路由标识为 R
Router#show ip route

// 2. 查看RIP协议运行详情、邻居、更新信息
Router#show ip protocols

// 3. 实时查看RIP路由更新过程（排错用）
Router#debug ip rip

// 4. 关闭debug调试
Router#no debug all
```

### 5. RIP 核心注意事项

1. `network` 命令**只写直连主网段**，RIP v1/v2 均不需要加子网掩码；
2. 全网所有路由器必须开启 RIP、宣告对应直连网段；
3. 最大跳数 15 跳，不适合大型广域网；
4. RIP v2 建议必加 `no auto-summary`，否则子网路由会汇总导致通信异常。

## 五、补充：交换机简要配置（二层傻瓜 / 网管交换机）

### 1. 二层交换机基础（仅局域网转发，无 IP 也可工作）

1. 普通傻瓜交换机：**即插即用**，无需任何配置，终端接网线即可同网段通信；
2. 网管型二层交换机（基础管理 IP 配置，仅用于远程管理）：

ios

```
Switch>enable
Switch#configure terminal
// 进入虚拟接口Vlan1（交换机默认管理VLAN）
Switch(config)#interface Vlan1
Switch(config-if)#ip address 192.168.16.100 255.255.255.0
Switch(config-if)#no shutdown
Switch(config-if)#exit

// 配置网关（用于跨网段远程管理）
Switch(config)#ip default-gateway 192.168.16.1
```

### 2. 交换机 & 路由器核心区别汇总

|设备|转发依据|核心功能|广播域|典型配置|
|---|---|---|---|---|
|二层交换机|MAC 地址|同网段终端转发|整个交换网络为 1 个广播域|仅配置管理 IP/VLAN|
|路由器|IP 地址|跨网段路由、网关|每个接口独立广播域|接口 IP、静态路由、RIP 动态路由|

# 一、Linux 网络相关全套命令（按运维常用排序）

1、网关、网卡

```bash
1.查看默认网关
	a.ip route
		ip route
		ip route|grep default #只显示默认网关
		ip route|grep default|awk '{print $3}' #仅提取网关IP
	
	b.route
		#需要安装
		yum -y install net-tools
		#用法
		route -n #0.0.0.0 通往所有ip，该行为网关

2.查看网卡IP
	a.ip address
	b.hostname -I

3.临时配置
	a.临时配置默认网关
		#添加默认网关(via 后接网关IP，dev 后接网卡名)
		ip route add default via 10.0.0.2 dev eth0
		route add default gw 10.0.0.2
		#删除默认网关
		ip route del default via 10.0.0.2
		route del default gw 10.0.0.2
	
	b.临时配置网卡IP
		#临时添加IP
		ip addr add 10.0.0.201/24 dev eth0
		#临时删除IP
		ip addr del 10.0.0.201/24 dev eth0
	
	c.永久修改配置
		vim /etc/sysconfig/network-scripts/ifcfg-eth0
	
	d.修改后需要重启网络服务
		systemctl restart network #重启网络服务
		#或单独重启网卡
		ifdown eth0 && ifup eth0
```

2、网络监听

```bash
1.ping 检测主机是否可达
	ping baidu.com
	ping -c 4 192.168.1.1    # -c 指定发包数量，发4次自动停止
	ping -i 0.5 192.168.1.1  # -i 发包间隔(秒)，默认1s
	ping -W 2 192.168.1.1    # -W 超时等待时间
	ping -s 1000 192.168.1.1 # -s 自定义数据包大小

2.telnet 端口连通性测试
	telnet 192.168.1.100 80

3.nc（netcat，网络瑞士军刀）
	a.端口检测
		nc -zv 192.168.1.100 22 # -z 只扫描不发送数据，-v 显示详情 
		nc -zv 192.168.1.100 80-90 # 扫描端口段 
		nc -uzv 192.168.1.100 53 # -u UDP端口探测(DNS)
	b.监听端口
		nc -l 8888 # 本机监听8888端口
	c.文件传输
		# 接收端 
		nc -l 9999 > test.tar.gz 
		# 发送端 
		nc 192.168.1.100 9999 < test.tar.gz

4.nmap 端口扫描，存活主机
	nmap 192.168.1.100               # 扫描常用1000个端口
	nmap -p 22,80,443 192.168.1.100  # 指定端口扫描
	nmap -p 1-1000 192.168.1.100     # 扫描端口范围
	nmap -sn 192.168.1.0/24          # 网段存活主机探测(不扫端口)
	nmap -sV 192.168.1.100           # 探测端口对应的服务版本
	nmap -T4 192.168.1.100           # -T 扫描速度，T4适中

5.netstat 端口查看
	netstat -lntp  # 最常用：监听端口
	# -l listen监听端口 -n 不解析域名/端口名 -t TCP -p 显示进程PID/程序名
	netstat -lnup  # UDP监听端口
	netstat -antp  # 所有TCP连接(已建立/监听/等待)
	netstat -i     # 网卡流量统计
	netstat -r     # 查看路由表

6.ss 高性能netstat
	ss -lntp    # 查看TCP监听端口
	ss -lnup    # UDP监听端口
	ss -antp    # 全部TCP连接
	ss -s       # 连接汇总统计(TCP总连接、TIME_WAIT数量)
	ss -ti      # 显示TCP内部信息（定时器、重传）
	ss state ESTABLISHED # 只看已建立连接
	ss state TIME-WAIT   # 只看等待关闭连接

7.traceroute 路由追踪
	traceroute www.baidu.com       # 默认UDP探测，部分防火墙拦截
	traceroute -I www.baidu.com    # -I 使用ICMP包探测，等价Windows tracert
	traceroute -T -p 80 www.baidu.com # -T TCP探测，指定80端口
	traceroute -w 3 www.baidu.com  # -w 每一跳超时等待3秒

8.iftop 实时网卡流量监控
	#需要安装
	yum install iftop
	#用法
	iftop          # 默认监控第一块网卡eth0
	iftop -i eth1  # 指定网卡
	iftop -n       # 只显示IP，不解析域名（更快）
	iftop -P       # 同时显示端口号
	#快捷键
	T      #显示总流量
	=> <=  #切换排序
	q      #退出

9.dstat 资源综合监控
	#需要安装
	yum -y install dstat
	#用法
	dstat                # 默认cpu、磁盘、网络、分页、系统
	dstat -n 1           # 只看网络流量，1秒刷新一次
	dstat -cdngy 1       # cpu+磁盘+网络+内存+swap+系统
	dstat --top-net      # 展示占用流量最高的进程
	dstat --top-io       # 磁盘IO最高进程
```
