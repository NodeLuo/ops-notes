
1、vim异常退出问题

```bash
vim 工作流程
	1.vim a.txt 进入视图模式
	2.系统生成一个隐藏文件 .a.txt.swp
	3.若正常保存退出，系统会将 .a.txt.swp 移动成 a.txt

vim 异常退出问题解决：
	1.保留修改后的内容
		a.输入 vim -r a.txt，后续直接回车
		b.输入 :wq 保存退出
		c.删除隐藏文件 .a.txt.swp
	
	2.不保留修改后的内容
		a.直接删除隐藏文件 .a.txt.swp


vim 异常退出原因：
	1.修改后直接关闭xshell
	2.修改后直接断电
	3.多个窗口同时编辑一个文件
	4.和同事同时编辑同一个文件
```

2、Linux重要系统配置文件

```bash
1.网卡配置文件
	文件位置：/etc/sysconfig/network-scripts/ifcfg-eth0
	
	[root@breadbomb ~]# cat /etc/sysconfig/network-scripts/ifcfg-eth0
	TYPE=Ethernet          #网络类型 以太网
	BOOTPROTO=none         #配置IP地址的方式 
								# none 表示自定义配置
								# static 和none效果一致
								# dhcp 表示自动获取ip
	NAME=eth0              #网卡名称
	DEVICE=eth0            #硬件名称
	ONBOOT=yes             #开机自动运行network服务
	IPADDR=10.0.0.200      #IP地址
	PREFIX=24              #子网掩码
	GATEWAY=10.0.0.2       #网关
	DNS1=223.5.5.5         #公用的DNS服务器地址
								# 223.5.5.5
								# 114.114.114.114
								# 8.8.8.8
	
	UUID=...   #物理地址，直接删除


2.网卡简化之后的检查操作
	a.重启网络服务（若无提示则修改正确）
		systemctl restart network
	b.检查ip地址
		ip a
	c.检查网络连接
		ping www.baidu.com
	d.
```
