
1、vim异常退出问题

```bash
vim 工作流程
	1.vim a.txt 进入视图模式
	2.系统生成一个隐藏文件 .a.txt.swp
	3.若正常保存退出，系统会将 .a.txt.swp 移动成 a.txt

vim 异常退出问题解决：
	1.保留修改后的内容
		a.输入 vim -r a.txt，后续直接回车
		b.输入 :wq 保存退出
		c.删除隐藏文件 .a.txt.swp
	
	2.不保留修改后的内容
		a.直接删除隐藏文件 .a.txt.swp


vim 异常退出原因：
	1.修改后直接关闭xshell
	2.修改后直接断电
	3.多个窗口同时编辑一个文件
	4.和同事同时编辑同一个文件
```

2、Linux重要系统配置文件

```bash
1.网卡配置文件
	文件位置：/etc/sysconfig/network-scripts/ifcfg-eth0
	
	[root@breadbomb ~]# cat /etc/sysconfig/network-scripts/ifcfg-eth0
	TYPE=Ethernet          #网络类型 以太网
	BOOTPROTO=none         #配置IP地址的方式 
								# none 表示自定义配置
								# static 和none效果一致
								# dhcp 表示自动获取ip
	NAME=eth0              #网卡名称
	DEVICE=eth0            #硬件名称
	ONBOOT=yes             #开机自动运行network服务
	IPADDR=10.0.0.200      #IP地址
	PREFIX=24              #子网掩码
	GATEWAY=10.0.0.2       #网关
	DNS1=223.5.5.5         #公用的DNS服务器地址
								# 223.5.5.5
								# 114.114.114.114
								# 8.8.8.8
	
	UUID=...   #物理地址，直接删除


2.网卡简化之后的检查操作
	a.重启网络服务（若无提示则修改正确）
		systemctl restart network
	b.检查ip地址
		ip a
	c.检查网络连接
		ping www.baidu.com
	d.检查DNS服务 本地 公网
		本地DNS服务：
			cat /etc/hosts
		公网DNS服务：
			cat /etc/resolv.conf

3./etc/hosts 本地DNS解析 本地域名解析服务（可实现网络劫持）
	[root@breadbomb ~]# cat /etc/hosts
	127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
	::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
	
	windows中的hosts文件位置：
		C:\windows\System32\drivers\etc\hosts
		修改方案：
			a.复制到桌面
			b.修改复制文件
			c.移动覆盖

4./etc/hostname 主机名存放位置
	[root@breadbomb ~]# cat /etc/hostname
	breadbomb
	
	查看主机名：
		hostname #直接回车
	
	注意：每次重启虚拟机系统会自动读取 /etc/hostname 中的主机名
	
	修改主机名方法：
		1.临时修改主机名，重启系统失效
			hostname 主机名
		2.永久修改主机名，需要重启生效
			vim /etc/hostname
		3.临时+永久修改主机名
			hostnamectl set-hostname 主机名

5./etc/resolv.conf DNS配置存放位置
	注意：
	虚拟机重启读取网卡配置文件中的DNS配置，覆盖当前文件内容
	若该文件中包含配置中DNS 或 网卡配置文件中没有配置DNS，则不变
	
	修改方式：
		vim /etc/resolv.conf
		
		[root@breadbomb ~]# cat /etc/resolv.conf
		# Generated by NetworkManager
		nameserver 223.5.5.5
		nameserver 114.114.114.114 # 可使用vim添加备用
	
	查看当前发挥作用的DNS地址：
		nslookup www.baidu.com
		
		安装nslookup:
			yum -y install bind-utils
		
		示例：
			[root@breadbomb ~]# nslookup www.baidu.com
			Server:		223.5.5.5
			Address:	223.5.5.5#53
			
			Non-authoritative answer:
			www.baidu.com	canonical name = www.a.shifen.com.
			Name:	www.a.shifen.com
			Address: 36.152.44.132
			Name:	www.a.shifen.com
			Address: 36.152.44.93
			Name:	www.a.shifen.com
			Address: 2409:8c20:6:1794:0:ff:b080:87f0
			Name:	www.a.shifen.com
			Address: 2409:8c20:6:123c:0:ff:b0f6:b2d
			
		网卡配置文件配置方案：
			DNS1=223.5.5.5
			DNS2=114.114.114.114

6./etc/profile 存储环境变量
	a.全局生效
	b.登录时加载，xshell重连就会执行一次

7./etc/fstab 开机自动挂载
```
