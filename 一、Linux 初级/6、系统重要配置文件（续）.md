
1、Linux系统重要配置文件

```bash
1./etc/rc.local  开机执行该文件中命令

	执行命令：给文件增加可执行权限
	chmod +x /etc/rc.local
	
	a.开机后只执行一次
	b.可以向文件中写入命令
		vim /etc/rc.local

2./etc/inittab 系统的运行级别

	a.查看系统的运行级别
		runlevel #直接回车
	
	b.系统的运行级别：
		0    关机
		1    单用户,忘记密码可以从这里找回
		2    多用户，不支持NFS 网络文件系统
		3    多用户，当前使用的命令行界面
		4    保留待开发
		5    图形界面 需提前安装
		6    重启
	
	c.进入对应级别：
		init 0 关机
		init 5 进入图形界面
		init 6 重启
	
	d.系统初始化配置：
		（1）关闭NetworkManager
			systemctl stop NetworkManager
			systemctl disable NetworkManager
		（2）创建快照

3./etc/motd 开机屏幕自动显示内容

	#
	#         ┌─┐       ┌─┐
	#      ┌──┘ ┴───────┘ ┴──┐
	#      │                 │
	#      │       ───       │
	#      │  ─┬┘       └┬─  │
	#      │                 │
	#      │       ─┴─       │
	#      │                 │
	#      └───┐         ┌───┘
	#          │         │
	#          │         │
	#          │         │
	#          │         └──────────────┐
	#          │                        │
	#          │                        ├─┐
	#          │                        ┌─┘
	#          │                        │
	#          └─┐  ┐  ┌───────┬──┐  ┌──┘
	#            │ ─┤ ─┤       │ ─┤ ─┤
	#            └──┴──┘       └──┴──┘
	#                神兽保佑
	#                代码无BUG!
	
	vim 块模式快捷键
		a.crtl+v   块模式，批量选中
		b.shift+v  批量行选中

4./etc/issue 和 /etc/issue.net  显示版本等信息

	直接清空内容：
		> /etc/issue
		> /etc/issue.net
```

2、proc下重要配置文件

```bash
1./proc/cpuinfo 查看cpu信息
2./proc/meminfo 查看内存信息
3./proc/loadavg 查看负载信息
4./proc/mounts  查看挂载信息

命令：
	1.lscpu 查看cpu信息
	
	[root@breadbomb ~]# lscpu
	Architecture:          x86_64
	CPU op-mode(s):        32-bit, 64-bit
	Byte Order:            Little Endian
	CPU(s):                1               #总核心数
	On-line CPU(s) list:   0               #核心列表，如果是4个，就是0-3
	Thread(s) per core:    1
	Core(s) per socket:    1
	Socket(s):             1               #cpu个数
	
	2.free -h 查看内存信息
	
	[root@breadbomb ~]# free -h
	          总大小        使用         剩余      共享内存   缓冲/缓存     可用空间
			  total        used        free      shared  buff/cache   available
	Mem:      1.9G        166M        1.3G        9.5M        481M        1.6G
	Swap:     2.0G          0B        2.0G
	
	3.df -h 查看磁盘信息
	
	[root@breadbomb ~]# df -h
	硬件名称         大小   使用  可用  使用百分比   挂载点
	Filesystem      Size  Used Avail Use%        Mounted on
	devtmpfs        980M     0  980M   0%        /dev
	tmpfs           991M     0  991M   0%        /dev/shm
	tmpfs           991M  9.6M  981M   1%        /run
	tmpfs           991M     0  991M   0%        /sys/fs/cgroup
	/dev/sda3        18G  2.0G   16G  12%        /
	/dev/sda1       197M  110M   88M  56%        /boot
	tmpfs           199M     0  199M   0%        /run/user/0
	
	4.uptime 查看系统负载
	
	[root@breadbomb ~]# uptime
	 08:31:21 up 22:29,  2 users,    load average: 0.00, 0.01, 0.05
	 当前时间     运行时长  登录用户数量   平均负载     1分钟  5分钟  15分钟
	 
	 系统负载：
		 衡量操作系统繁忙程度
	
	负载和cpu核心数量相同：
		系统繁忙
		核心 4  负载接近或超过2 代表非常繁忙
	
	负载高：
		可能代表资源利用率高
	
	5.w 查看系统负载和用户登录信息
	
	[root@breadbomb ~]# w

	USER     TTY      FROM         LOGIN@   IDLE   JCPU   PCPU WHAT
	root     tty1                  Wed12    4:23   0.91s  0.91s -bash
	root     pts/0    10.0.0.1     08:17    7.00s  0.08s  0.00s w
```
