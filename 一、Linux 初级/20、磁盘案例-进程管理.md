
1、案例

```bash
1.由于进程占用大量内存导致系统出现oom报错信息
	oom内存溢出:
		导致系统杀死占用内存最高的进程
	解决办法：
		增加内存
	导致内存飙升的因素：
		a.由于大量用户访问           正常现象
		b.开发写的程序导致错误       找开发解决问题
		c.由于网络波动导致内存升高    增加swap容量解决
		d.其他原因 内存测试 拷贝 压力测试等
	总结：
		外部   增加内存
		内部   代码  个人操作  使用swap解决
		swap:用磁盘划分的分区充当内存使用(速度较慢)
			(1)swap可以在安装操作系统的时候进行划分
			(2)swap可以在安装完操作系统后划分
			(3)swap的大小是内存的1-1.5倍，若内存大于8G，则最多给8G

磁盘分区：
/boot   /    /swap
企业分区：
/boot   /
---------------------------------------------------------------------------------
查看swap分区大小：
[root@breadbomb ~]# free -h
              total        used        free      shared  buff/cache   available
Mem:           1.9G        174M        1.3G         17M        516M        1.6G
Swap:          2.0G          0B        2.0G

在系统中创建swap分区：
	a.生成一个2G空间的文件
		[root@breadbomb ~]#dd if=/dev/zero of=/tmp/2g bs=1M count=2000
	
	b.格式化为swap分区
		[root@breadbomb ~]#mkswap /tmp/2g
	
	c.激活swap空间(挂载到原swap之上)
		[root@breadbomb ~]#swapon /tmp/2g
		swapon -s #查看swap组成情况
	
	d.查看是否挂载成功(原来是2G，现在是4G)
	
	e.卸载分区
		[root@breadbomb ~]#swapoff /tmp/2g

查看swap分区大小：
[root@breadbomb ~]# free -h
              total        used        free      shared  buff/cache   available
Mem:           1.9G        174M        1.3G         17M        516M        1.6G
Swap:          4.0G          0B        2.0G
---------------------------------------------------------------------------------

2.磁盘满，如何查找大文件
	a.find查找
		[root@breadbomb ~]#find / -type f -size +100M
	b.du查看
		[root@breadbomb ~]#du -h /|awk '$1~/G$/'
		#继续查找大目录下的文件
		[root@breadbomb ~]#du -h /tmp/*|awk '$1~/G$/'

3.inode满，如何查找小文件
	a.find查找
		[root@breadbomb ~]#find / -type d -size +1M
		
		提示：目录存放的是下级目录和文件的名称，所以文件越多，目录越大
		#查看目录大小
		[root@breadbomb ~]# touch {1..1000}.txt
		[root@breadbomb ~]# ll -dh
		dr-xr-x---. 2 root root 28K Jun 14 10:51 .

4.磁盘满，通过增加磁盘的方式解决
	a.创建挂载点
		[root@breadbomb ~]#mkdir /data
	b.挂载大磁盘
		[root@breadbomb ~]#mount /dev/sdb /data
	c.移动日志文件
		[root@breadbomb ~]#mv /var/log/1.log /data/1.log
	d.软链接
		[root@breadbomb ~]#ln -s /data/1.log /var/log/1.log

```

```bash
5.文件被占用，磁盘空间不释放
	a.生成一个10G的大文件
		[root@breadbomb ~]#dd if=/dev/zero of=/data/1.log bs=1M count=10000
	b.模拟文件被调用(第一个窗口)
		[root@breadbomb ~]#tail /data/1.log
	c.删除文件(第二个窗口)
		[root@breadbomb ~]#rm -rf /data/1.log
	
	#此时空间依然被占用，未被释放
	
	d.通过lsof命令查找被那个进程所调用
		默认没有lsof命令
		需要安装：yum -y install lsof
		
		[root@breadbomb ~]#lsof|grep 1.log
	
	e.结束进程
		[root@breadbomb ~]#kill 4734 (该进程编号4734)
		#若是在线服务，需要重载，不能删除
		[root@breadbomb ~]#systemctl reload 服务
```

2、进程管理

```bash
1.查看当前进程状态
	ps -auxf 或 ps auxf
		-a 显示所有终端机下执行的进程，除阶段作业领导者之外
		-u 以用户为主的格式显示进程状况
		-x 显示所有进程，不以终端机区分
		-f 用ASCII字符显示树状结构，表达进程之间的相互关系
	
	[root@breadbomb ~]# ps auxf|head -2
	启动用户 进程ID cpu占用 内存占用 虚拟占用 实际占用 启动终端   状态 启动时间 占用cpu时间   进程名
	USER     PID   %CPU   %MEM    VSZ     RSS     TTY      STAT START   TIME COMMAND
	root      2    0.0    0.0      0       0       ?        S    Jun11   0:00 [kthreadd]
	
	STAT状态类型：
		a.S  可中断睡眠
		b.Ss 父进程
		c.S< 高优先级
		d.SN 低优先级
```
