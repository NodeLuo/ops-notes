
1、案例

```bash
1.由于进程占用大量内存导致系统出现oom报错信息
	oom内存溢出:
		导致系统杀死占用内存最高的进程
	解决办法：
		增加内存
	导致内存飙升的因素：
		a.由于大量用户访问           正常现象
		b.开发写的程序导致错误       找开发解决问题
		c.由于网络波动导致内存升高    增加swap容量解决
		d.其他原因 内存测试 拷贝 压力测试等
	总结：
		外部   增加内存
		内部   代码  个人操作  使用swap解决
		swap:用磁盘划分的分区充当内存使用(速度较慢)
			(1)swap可以在安装操作系统的时候进行划分
			(2)swap可以在安装完操作系统后划分
			(3)swap的大小是内存的1-1.5倍，若内存大于8G，则最多给8G

磁盘分区：
/boot   /    /swap
企业分区：
/boot   /
---------------------------------------------------------------------------------
查看swap分区大小：
[root@breadbomb ~]# free -h
              total        used        free      shared  buff/cache   available
Mem:           1.9G        174M        1.3G         17M        516M        1.6G
Swap:          2.0G          0B        2.0G

在系统中创建swap分区：
	a.生成一个2G空间的文件
		[root@breadbomb ~]#dd if=/dev/zero of=/tmp/2g bs=1M count=2000
	
	b.格式化为swap分区
		[root@breadbomb ~]#mkswap /tmp/2g
	
	c.激活swap空间(挂载到原swap之上)
		[root@breadbomb ~]#swapon /tmp/2g
		swapon -s #查看swap组成情况
	
	d.查看是否挂载成功(原来是2G，现在是4G)
	
	e.卸载分区
		[root@breadbomb ~]#swapoff /tmp/2g

查看swap分区大小：
[root@breadbomb ~]# free -h
              total        used        free      shared  buff/cache   available
Mem:           1.9G        174M        1.3G         17M        516M        1.6G
Swap:          4.0G          0B        2.0G
---------------------------------------------------------------------------------

2.磁盘满，如何查找大文件
	a.find查找
		[root@breadbomb ~]#find / -type f -size +100M
	b.du查看
		[root@breadbomb ~]#du -h /|awk '$1~/G$/'
		#继续查找大目录下的文件
		[root@breadbomb ~]#du -h /tmp/*|awk '$1~/G$/'

3.inode满，如何查找小文件
	a.find查找
		[root@breadbomb ~]#find / -type d -size +1M
		
		提示：目录存放的是下级目录和文件的名称，所以文件越多，目录越大
		#查看目录大小
		[root@breadbomb ~]# touch {1..1000}.txt
		[root@breadbomb ~]# ll -dh
		dr-xr-x---. 2 root root 28K Jun 14 10:51 .

4.磁盘满，通过增加磁盘的方式解决
	a.创建挂载点
		[root@breadbomb ~]#mkdir /data
	b.挂载大磁盘
		[root@breadbomb ~]#mount /dev/sdb /data
	c.移动日志文件
		[root@breadbomb ~]#mv /var/log/1.log /data/1.log
	d.软链接
		[root@breadbomb ~]#ln -s /data/1.log /var/log/1.log

```

```bash
5.文件被占用，磁盘空间不释放
	a.生成一个10G的大文件
		[root@breadbomb ~]#dd if=/dev/zero of=/data/1.log bs=1M count=10000
	b.模拟文件被调用(第一个窗口)
		[root@breadbomb ~]#tail /data/1.log
	c.删除文件(第二个窗口)
		[root@breadbomb ~]#rm -rf /data/1.log
	
	#此时空间依然被占用，未被释放
	
	d.通过lsof命令查找被那个进程所调用
		默认没有lsof命令
		需要安装：yum -y install lsof
		
		[root@breadbomb ~]#lsof|grep 1.log
	
	e.结束进程
		[root@breadbomb ~]#kill 4734 (该进程编号4734)
		#若是在线服务，需要重载，不能删除
		[root@breadbomb ~]#systemctl reload 服务
```

2、进程管理

```bash
1.查看当前进程状态
	ps -auxf 或 ps auxf
		-a 显示所有终端机下执行的进程，除阶段作业领导者之外
		-u 以用户为主的格式显示进程状况
		-x 显示所有进程，不以终端机区分
		-f 用ASCII字符显示树状结构，表达进程之间的相互关系
	
	top 动态查看进程
	
	[root@breadbomb ~]# ps auxf|head -2
	启动用户 进程ID cpu占用 内存占用 虚拟占用 实际占用 启动终端   状态 启动时间 占用cpu时间   进程名
	USER     PID   %CPU   %MEM    VSZ     RSS     TTY      STAT START   TIME COMMAND
	root      2    0.0    0.0      0       0       ?        S    Jun11   0:00 [kthreadd]
```

| **`标志`**      | **`意义`**                                                                |
| ------------- | ----------------------------------------------------------------------- |
| **`USER`**    | **`该 process 所属的用户账号`**                                                 |
| **`PID`**     | **`该 process 的进程 ID 号`**                                                |
| **`%CPU`**    | **`该 process 占用 CPU 资源的百分比`**                                           |
| **`%MEM`**    | **`该 process 占用物理内存的百分比`**                                              |
| **`VSZ`**     | **`该 process 使用的虚拟内存总量（单位 Kbytes）`**                                    |
| **`RSS`**     | **`该 process 占用的常驻物理内存量（单位 Kbytes）`**                                   |
| **`TTY`**     | **`运行该 process 对应的终端；无关联终端显示?；tty1-tty6 为本机终端登录进程，pts/0 等为远程网络连接终端进程`** |
| **`STAT`**    | **`当前进程运行状态`**                                                          |
| **`START`**   | **`process 启动的时间`**                                                     |
| **`TIME`**    | **`process 实际占用 CPU 运行的累计时间`**                                          |
| **`COMMAND`** | **`启动该进程的实际指令；带[]为内核进程，无[]为用户态进程`**                                     |

| **`STAT 基本状态`** | **`描述`**           | **`STAT 状态 + 符号`** | **`描述`**                         |
| ----------- | -------------- | -------------- | ---------------------------- |
| **`R`**         | **`进程运行 / 可就绪调度`** | **`s`**            | **`进程是会话首进程，Ss 代表可中断睡眠的会话领导进程`** |
| **`S`**         | **`可中断睡眠`**        | **`<`**            | **`进程运行在高优先级上，S < 为优先级较高的进程`**   |
| **`T`**         | **`进程被暂停`**        | **`N`**            | **`进程运行在低优先级上，SN 为优先级较低的进程`**    |
| **`D`**         | **`不可中断睡眠`**       | **`+`**            | **`当前进程关联终端前台，R + 表示该进程在前台运行`**  |
| **`Z`**         | **`僵尸进程`**         | **`l`**            | **`进程是多线程的，Sl 表示进程以多线程方式运行`**    |
| **`X`**         | **`死亡进程`**         | **`L`**            | **`进程持有内存锁定页面`**                 |
```bash
状态模拟(两个终端，一个模拟，另一个查询)
	#运行
	[root@breadbomb ~]#vim test.txt
	
	#查询test.txt相关进程
	[root@breadbomb ~]#ps -aux|grep test.txt
	root      46473  0.9  0.2 149508  5208 pts/2    S+   14:49   0:00 vim test.txt
	root      46487  0.0  0.0 112808   976 pts/3    R+   14:50   0:00 grep --color=auto test.txt    #这一行是grep命令本身的进程
	
	#省略grep当前进程
	[root@breadbomb ~]# ps -aux|grep test.txt|grep -v grep
	root      46473  0.0  0.2 149508  5208 pts/2    S+   14:49   0:00 vim test.txt
	
	#挂起，在vim test.txt中按crtl+z
	[root@breadbomb ~]# ps -aux|grep test.txt|grep -v grep
	root      46473  0.0  0.2 149508  5208 pts/2    T    14:49   0:00 vim test.txt
	
	#在压缩打包的时候，由S+,R+变成D+
	[root@breadbomb ~]# ps -aux|grep tar|grep -v grep
	root      46500  0.0  0.1 134568  3124 pts/2    D+   14:52   0:02 tar -zcf bigfile.tar.gz bigfile
```

| **`任务`**               | **`含义`**                        |
| ---------------- | ------------------------- |
| **`Tasks:73 total`**   | **`当前进程的总数`**                   |
| **`2 running`**        | **`正在运行的进程数`**                  |
| **`71 sleeping`**      | **`睡眠的进程数`**                    |
| **`0 stopped`**        | **`停止的进程数`**                    |
| **`0 zombie`**         | **`僵尸进程数`**                     |
| **`%Cpu(s): 49.2 us`** | **`系统用户进程使用 CPU 百分比`**          |
| **`5.7 sy`**           | **`内核进程占用 CPU 百分比，内核是于硬件进行交互`** |
| **`ni`**               | **`改变过优先级的进程占用 CPU 百分比`**       |
| **`45.2 id`**          | **`空闲 CPU 的百分比`**               |
| **`0.0 wa`**           | **`CPU 等待 IO 完成的时间`**           |
| **`0.0 hi`**           | **`硬中断，占的 CPU 百分比`**            |
| **`0.0 si`**           | **`软中断，占的 CPU 百分比`**            |
| **`0.0 st`**           | **`比如虚拟机占用物理 CPU 的时间`**         |

软中断和硬中断的区别

|                  | **`软中断`**      | **`硬中断`**      |
| ---------------- | -------------- | -------------- |
| **`是否有随机性 突发性`** | **`否`**        | **`是`**        |
| **`是否有中断响应周期`**  | **`无`**        | **`是`**        |
| **`中断类型号的提供方法`** | **`固定或由指令提供`** | **`由中断控制器提供`** |
