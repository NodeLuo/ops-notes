3、RAID阵列

```bash
1.可以增加使用空间
	
	需求：1.2T
	库存：900G*8块
	方法：用RAID逻辑的将多个盘组合成一个盘，900G*2块=1.8T

2.实现安全冗余（备份）
	需求：400G
	库存：900G*8块
	方法：用RAID逻辑的将两个盘合成一个盘，一个用于存数据，另一个用于备份，900G*2=900G

3.提高读写速度（IO）
	使用多个盘实现并发IO，提升数据读写速度


笔试题：
写出RAID0 RAID1 RAID5 RAID10的区别：

RAID级别  最少需要盘数    安全性           容量        性能     使用场景
RAID0        1    坏一块所有数据都损坏  所有盘的总和   读写快   只要性能不要安全（存缓存）
RAID1      只能2块    可以损坏一块盘       50%      写慢读可以 只要安全不要速度（监控）
RAID5      至少3块    可以损坏一块盘       n-1      读写都可以  稳定的业务
RAID10 （偶数）至少4块  可以损坏一半        一半       读写快    高并发业务场景
```

4、磁盘分区

```bash
1.MBR格式最多支持4个分区
	C D E F
2.MBR格式支持3个主分区+1个扩展分区
3.基于扩展分区可以划分逻辑分区
只有主分区和逻辑分区可以正常写入数据



分区命令：
	大于2T的磁盘使用 parted 分区  128个主分区
	小于2T的磁盘使用 fdisk  分区  4个主分区


企业中磁盘不够用需要添加新的磁盘：
	虚拟机添加流程：
		（1）插入一块虚拟磁盘
			启动虚拟机-->右键设置-->点击硬盘-->添加硬盘-->选择磁盘类型-->填入大小-->完成
		（2）重启系统生效
			a.虚拟机需要重启
			b.物理机不需要重启（支持热插拔）
			c.云服务器扩容，支持在线扩容
		（3）查看硬盘
			fdisk -l   #sdb表示第二块硬盘
		（4）开始分区
			fdisk /dev/sdb #进入磁盘分区界面
			
			Command (m for help): m
			Command action
			   a   toggle a bootable flag                
			   b   edit bsd disklabel                    
			   c   toggle the dos compatibility flag     
			   d   delete a partition                    # 删除分区
			   F   list free unpartitioned space         
			   l   list known partition types            
			   n   add a new partition                   # 新建分区
			   o   create a new empty DOS partition table
			   p   print the partition table             # 打印查看分区表
			   q   quit without saving changes           # 退出不保存
			   s   create a new empty Sun disklabel      
			   t   change a partition''s system id        
			   u   change display/entry units           
			   v   verify the partition table            
			   w   write table to disk and exit          # 保存分区表并退出
			   x   extra functionality (experts only)    
			
			
			Command (m for help): n                       #添加新分区
			Partition type 
				p primary (0 primary, 0 extended, 4 free) #主分区
				e extended                                #扩展分区
				Select (default p):                       #默认回车主分区
			Partition number (1-4, default 1):            #默认分区编号
			First sector (xxxx-xxxx, default xxxx):       #默认开始分区
			Last sector, +/-sectors or +/-size{K,M,G,T,P} (xxxx-xxxx, default xxxx):+1G                                                    #结束大小手动配置
		
		（5）格式化分区（装修打扫卫生,格式化之后才可以写入数据）
			mkfs.xfs /dev/sdb1
			
			mkfs.*** (后面的类型根据需求选，没需求默认xfs)
		
		（6）挂载使用
			a.创建入口目录，空目录
				mkdir /a
				mkdir /b
			b.挂载
				mount /dev/sdb1 /a
				mount /dev/sdb5 /b
			c.查看磁盘分区
				df -h
		
		（7）开机自动挂载
			a.查看磁盘UUID
				blkid
			b.开机挂载文件 /etc/fstab
				第一列可以写UUID，也可以直接使用硬件名称/dev/sdb1
				/dev/sdb1 /mnt xfs defaults 0 0
	
	企业使用磁盘的方式：
	1.插入磁盘
	2.格式化磁盘
		a.若卸载后需要使用参数-f强制格式化
		b.要卸载完所有的分区才可以统一格式化
			mkfs.xfs -f /dev/sdb
	3.挂载磁盘
		mount /dev/sdb /mnt
	4.写入开机自动挂载
		vim /etc/fstab
		
		/dev/sdb   /mnt    xfs    default  0 0
```

5、parted分区

```bash
GPT格式：大于2T的磁盘进行分区
	1.插入一个大于2T的磁盘
		[root@breadbomb ~]#fdisk -l  #查看分区
	2.分区
		[root@breadbomb ~]#parted /dev/sdb  #开始分区
		help           #显示菜单
		mklabel        #制作新标签
		mkpart         #创建分区
		print          #打印分区表
		quit           #退出
		rm NUMBER      #删除分区
		
		执行步骤：
			a.mklabel GPT
			b.mkpart 
				name: primary      #分区名 
				type: xfs          #类型
				start: 0           #开始
				end: 10G           #结束
				ignore/Cancel: I   #分区位置警告
	3.格式化
	4.挂载使用
	5.写入开机自动挂载
```
