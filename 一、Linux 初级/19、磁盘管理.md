3、RAID阵列

```bash
1.可以增加使用空间
	
	需求：1.2T
	库存：900G*8块
	方法：用RAID逻辑的将多个盘组合成一个盘，900G*2块=1.8T

2.实现安全冗余（备份）
	需求：400G
	库存：900G*8块
	方法：用RAID逻辑的将两个盘合成一个盘，一个用于存数据，另一个用于备份，900G*2=900G

3.提高读写速度（IO）
	使用多个盘实现并发IO，提升数据读写速度


笔试题：
写出RAID0 RAID1 RAID5 RAID10的区别：

RAID级别  最少需要盘数    安全性           容量        性能     使用场景
RAID0        1    坏一块所有数据都损坏  所有盘的总和   读写快   只要性能不要安全（存缓存）
RAID1      只能2块    可以损坏一块盘       50%      写慢读可以 只要安全不要速度（监控）
RAID5      至少3块    可以损坏一块盘       n-1      读写都可以  稳定的业务
RAID10 （偶数）至少4块  可以损坏一半        一半       读写快    高并发业务场景
```

4、磁盘分区

```bash
1.MBR格式最多支持4个分区
	C D E F
2.MBR格式支持3个主分区+1个扩展分区
3.基于扩展分区可以划分逻辑分区
只有主分区和逻辑分区可以正常写入数据



分区命令：
	大于2T的磁盘使用 parted 分区  128个主分区
	小于2T的磁盘使用 fdisk  分区  4个主分区


企业中磁盘不够用需要添加新的磁盘：
	虚拟机添加流程：
		（1）插入一块虚拟磁盘
			启动虚拟机-->右键设置-->点击硬盘-->添加硬盘-->选择磁盘类型-->填入大小-->完成
		（2）重启系统生效
			a.虚拟机需要重启
			b.物理机不需要重启（支持热插拔）
			c.云服务器扩容，支持在线扩容
		（3）查看硬盘
			fdisk -l   #sdb表示第二块硬盘
		（4）开始分区
			fdisk /dev/sdb #进入磁盘分区界面
			
			Command (m for help): m
			Command action
			   a   toggle a bootable flag                # 切换分区可启动标志
			   b   edit bsd disklabel                    # 编辑BSD磁盘标签
			   c   toggle the dos compatibility flag     # 切换DOS兼容标志
			d   delete a partition                    # 删除分区
			   F   list free unpartitioned space         # 列出空闲未分区空间
			   l   list known partition types            # 查看所有分区类型ID
			   n   add a new partition                   # 新建分区
			   o   create a new empty DOS partition table# 创建空DOS分区表
			   p   print the partition table             # 打印查看分区表
			   q   quit without saving changes           # 退出不保存
			   s   create a new empty Sun disklabel      # 创建空Sun磁盘标签
			   t   change a partition's system id        # 修改分区类型ID
			   u   change display/entry units           # 切换显示单位
			   v   verify the partition table            # 校验分区表
			   w   write table to disk and exit          # 保存分区表并退出
			   x   extra functionality (experts only)    # 专家高级功能

```
