
1、特殊符号

```bash
1.<< 标准输入重定向
	示例：
		#追加文件内容而不使用vim
		cat >> a.txt << EOF （EOF 可以自定义，结束符）
		
		[root@breadbomb ~]# cat >> a.txt << EOF
        > hello
        > 你好
        > EOF
        [root@breadbomb ~]# cat a.txt
		hello
		你好
		
		#输出到屏幕
		cat << EOF
		
		[root@breadbomb ~]# cat << eof
        > hello
        > 你好
        > back
        > eof
		hello
		你好
		back

2. '' 和 ""
	单引号不会识别特殊内容（正则可以）
	echo '$LANG' ----> $LANG
	双引号会识别特殊内容
	echo "$LANG" ----> en-US.UTF-8
```

2、正则表达式

```bash
三剑客：grep sed awk

	grep -n 显示过滤到的内容的行
	grep -o 只显示过滤后的内容，每次匹配一个
	
	示例：
		# grep -n
		[root@breadbomb ~]# grep -n 'l' a.txt 
		1:hello 
		2:world	
		
		# grep -o 
		#每次匹配一个，所以一行只有一个字符
		#这里跟匹配的内容长度有关，l是单个
		[root@breadbomb ~]# grep -o 'l' a.txt
		l
		l
		l



1.^ 以...开头

#以h开头
[root@breadbomb ~]# grep '^h' a.txt
hello

#以 h 或 你 开头
[root@breadbomb ~]# grep '^[h你]' a.txt
hello
你好

2.$ 以...结尾

#以o结尾
[root@breadbomb ~]grep 'o$' a.txt
hello

a.结尾有时候看不到空格：
	cat -A 查看每一行以什么结尾
	（1）^I 表示tab结尾
	（2）^$ 表示当前行为空行

b.$在前面表示变量，在双引号中会解析变量的值
	[root@breadbomb ~]# grep '$LANG' a.txt
	
	[root@breadbomb ~]# grep "$LANG" a.txt
	en_US.UTF-8


3. .（点）表示任意一个字符

a.任意一个字符过滤不会匹配到空行
	[root@breadbomb ~]# grep -o '.' a.txt
	hello 
	world	
	en_US.UTF-8

4. * 前一个字符出现0次或0次以上

#有0个或0个以上的l
[root@breadbomb ~]# grep 'l*' a.txt
hello 
world	
en_US.UTF-8
注意：因为*前面的l可以是0个，所以en_US.UTF-8也算


5. .* 贪婪匹配，匹配所有

#任意匹配
[root@breadbomb ~]# grep -o '.*' a.txt
hello 
world	
en_US.UTF-8

#任意长度开头，必须以l结尾
[root@breadbomb ~]# grep -o '^.*l' a.txt
hell
worl

#以l开头，任意结尾
[root@breadbomb ~]# grep -o 'l.*$' a.txt
llo 
ld	

6. [] 任意取一，或者的关系

[root@breadbomb ~]# grep [hd] a.txt
hello 
world	
[root@breadbomb ~]# grep -o [hd] a.txt
h
d
```
