1、特殊符号

```bash
1.<< 标准输入重定向
	示例：
		#追加文件内容而不使用vim
		cat >> a.txt << EOF （EOF 可以自定义，结束符）
		
		[root@breadbomb ~]# cat >> a.txt << EOF
        > hello
        > 你好
        > EOF
        [root@breadbomb ~]# cat a.txt
		hello
		你好
		
		#输出到屏幕
		cat << EOF
		
		[root@breadbomb ~]# cat << eof
        > hello
        > 你好
        > back
        > eof
		hello
		你好
		back

2. '' 和 ""
	单引号不会识别特殊内容（正则可以）
	echo '$LANG' ----> $LANG
	双引号会识别特殊内容
	echo "$LANG" ----> en-US.UTF-8
```

2、正则表达式

|   功能    | 基础正则（必须加\） |  扩展正则   |
| :-----: | :--------: | :-----: |
| 0 或 1 次 |    `\?`    |   `?`   |
| 1 次或多次  |    `\+`    |   `+`   |
| 或（多选一）  |    `\|`    |   \|    |
|   分组    |  `\( \)`   |  `( )`  |
|  精确次数   | `\{n,m\}`  | `{n,m}` |

```bash
三剑客：grep sed awk

	grep -n              显示过滤到的内容的行
	grep -o              只显示过滤后的内容，每次匹配一个
	grep -v              取反
	grep -E 或 egrep     扩展正则
	
	扩展正则 和 基础正则 区分：
		（1）基础正则需要 \ ,比如 \+ 表示一个或多个
			[root@breadbomb ~]# grep 'l\+' a.txt
			hello 
			world	
			
		（2）扩展正则不需要\
			[root@breadbomb ~]# egrep 'l+' a.txt
			hello 
			world	
		
	
	
	示例：
		# grep -n
		[root@breadbomb ~]# grep -n 'l' a.txt 
		1:hello 
		2:world	
		
		# grep -o 
		#每次匹配一个，所以一行只有一个字符
		#这里跟匹配的内容长度有关，l是单个
		[root@breadbomb ~]# grep -o 'l' a.txt
		l
		l
		l
		
		# grep -v
		[root@breadbomb ~]#  grep -v 'hello' a.txt
		world	
		en_US.UTF-8




1.^ 以...开头

	#以h开头
	[root@breadbomb ~]# grep '^h' a.txt
	hello
	
	#以 h 或 你 开头
	[root@breadbomb ~]# grep '^[h你]' a.txt
	hello
	你好

2.$ 以...结尾

	#以o结尾
	[root@breadbomb ~]grep 'o$' a.txt
	hello
	
	a.结尾有时候看不到空格：
		cat -A 查看每一行以什么结尾
		（1）^I 表示tab结尾
		（2）^$ 表示当前行为空行
	
	b.$在前面表示变量，在双引号中会解析变量的值
		[root@breadbomb ~]# grep '$LANG' a.txt
		
		[root@breadbomb ~]# grep "$LANG" a.txt
		en_US.UTF-8


3. .（点）表示任意一个字符

	a.任意一个字符过滤不会匹配到空行
		[root@breadbomb ~]# grep -o '.' a.txt
		hello 
		world	
		en_US.UTF-8

4. * 前一个字符出现0次或0次以上

	#有0个或0个以上的l
	[root@breadbomb ~]# grep 'l*' a.txt
	hello 
	world	
	en_US.UTF-8
	注意：因为*前面的l可以是0个，所以en_US.UTF-8也算


5. .* 贪婪匹配，匹配所有

	#任意匹配
	[root@breadbomb ~]# grep -o '.*' a.txt
	hello 
	world	
	en_US.UTF-8

	#任意长度开头，必须以l结尾
	[root@breadbomb ~]# grep -o '^.*l' a.txt
	hell
	worl
	
	#以l开头，任意结尾
	[root@breadbomb ~]# grep -o 'l.*$' a.txt
	llo 
	ld	

6. [] 任意取一，或者的关系

	[root@breadbomb ~]# grep [hd] a.txt
	hello 
	world	
	[root@breadbomb ~]# grep -o [hd] a.txt
	h
	d

	注意：中括号中特殊符号都失效 ，^在中括号中表示取反,需要 \^ 转义
		[root@breadbomb ~]# cat a.txt
		hello 
		world	
		en_US.UTF-8
		.*?^$!;: 特殊符号
		[root@breadbomb ~]# grep '[a-zA-Z0-9.*?\^$!;:]' a.txt
		hello 
		world	
		en_US.UTF-8
		.*?^$!;: 特殊符号


7. + 出现一次或一次以上
   
	a.基础正则需要 \+
	b.扩展正则
		egrep 或 grep -E
		
	示例：
	[root@breadbomb ~]# egrep 'l+' a.txt
	hello 
	world	

8. | 或者 

	a.基础正则需要 \|
	b.扩展正则
		egrep 或 grep -E
	
	示例：
	[root@breadbomb ~]# grep -E 'e|d' a.txt
	hello 
	world	
	en_US.UTF-8

9. {} 范围 该符号前面的字符出现几次

	a.基础正则需要 \{ \}
	b.扩展正则
		egrep 或 grep -E
	
	种类：
	（1）{m,} 大于等于m次
	（2）{m,n} 大于等于m,小于等于n次
	（3）{,n} 小于等于n次
	（4）{n} 恰好是n次
	
	示例：
	[root@breadbomb ~]# grep -E 'l{1,}' a.txt
	hello 
	world	



```
