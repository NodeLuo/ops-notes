
1、压缩

```bash
1.tar 命令 （默认打包到当前位置）
	-z 使用gzip压缩
	-c 压缩
	-x 解压 （解压的时候要注意覆盖文件的安全问题）
	-v 显示过程
	-f 指定压缩包文件名
	-C 切换压缩路径、指定解压位置
	-t 查看压缩包中的文件
	tar -zcvf 名称.tar.gz 文件1 文件2 文件3
	
	压缩的目的：
		a.节省磁盘空间
		b.节省传输流量
		c.减少磁盘IO，降低服务器负载
	
	压缩的时机：
		a.业务的低谷期
	
	示例：
		#多文件压缩、目录
		tar -zcvf all.tar.gz a.txt b.txt example/
		
		#压缩位置
		tar -zcvf /opt/a.tar.gz a.txt
		
		#排除某个文件再压缩
			--exclude 排除
			--exclude-from 排除文件列表（将需要排除的文件或目录都放在一个文件中）
			
				当前目录下有：{1..5}.txt one.txt haha.txt
				a.排除1.txt one.txt haha.txt
					tar -zcvf all.tar.gz *.txt --exclude={1,one,haha}.txt
				b.排除1.txt one.txt
					ls 1.txt one.txt > ex.txt
					tar -zcvf all.tar.gz *.txt --exclude-from=ex.txt
		
		#安全提示 (系统为了文件安全压缩默认删除/目录)
		tar -zcvf passwd.tar.gz /etc/passwd
			
			[root@breadbomb ~]# tar -zcvf passwd.tar.gz /etc/passwd
			tar: Removing leading / from member names
			/etc/passwd
		
		
		tar -zcvf log.tar.gz /home/user/data/log.txt
		压缩结果：
			home/ - user/ - data/ - log.txt
		
		tar -zcvf log.tar.gz -C /home/user/data log.txt
		压缩结果：
			log.txt
		
		
		#解压到当前目录
		tar -zxvf a.tar.gz
		
		#解压到指定目录
		tar -zxvf a.tar.gz -C /tmp/

2.zip 和 unzip命令（需要安装）
	安装：
		yum -y install zip unzip
		
	zip 压缩
		-r 递归压缩
		
		#压缩test目录
		zip -r test.zip test/ 
	
	unzip 解压
		-d 指定目录
		unzip test.zip 
		
		# 解压到指定目录 
		unzip test.zip -d /opt/
```

2、文件权限

```bash
[root@breadbomb ~]# ll -i example
total 4
51262230 -rw-r--r-- 1 root root 12 May 14 09:54 a.txt

第三列：rw-r--r-- 9位文件权限
	完整权限：rwx
		r 读
		w 写
		x 可执行，脚本给予执行权限
		- 无权限
	
	
	一个sh文件的执行：
		（1）使用sh命令执行，可以直接执行（放在解释器中运行）
		（2）直接使用文件名执行 ./a.sh,此时需要x权限才可以执行

	作用：
		a.前三位 属主
		b.中三位 属组
		c.后三位 其他位置
		
		解释：系统创建用户是，会默认创建和用户名一致的组
			 root 用户 root 组
	
	文件默认权限：rw-r--r--
	目录默认权限：rwxr-xr-x
```

3、硬链接

```bash
[root@breadbomb ~]# ll -i example
total 4
51262230 -rw-r--r-- 1 root root 12 May 14 09:54 a.txt

第四列：硬链接 1 表示硬链接个数
	1.特点：
		a.具有相同Inode号码的文件互为硬链接
		b.不占磁盘空间
		c.只能跨同文件系统
		d.不支持主动给目录、不存在文件、特殊文件加硬链接
		e.删除一个硬链接，不影响其他，所有硬链接删除，文件被删
		f.修改一个硬链接，其他同步变化
		
		目录默认有两个硬链接，最底层的目录：目录本身 和 .
		例如：
			/root/example --->  example 和 .
			此时root有三个： root 和 . 和 ..
	
	2.创建硬链接
		ln 源文件 硬链接文件名
		
		验证方法：
		ls -li 查看inode号码是否一致
	
	3.作用
		a.实时备份，防止文件误删
		b.节省磁盘空间，硬链接共享一份数据
		c.系统底层目录结构依赖

```

4、软链接

```bash
1.特点
	a.inode号码不同
	b.相当于快捷方式
	c.软链接可以跨文件系统，但需要绝对路径
	d.删除源文件，软链接失效（变红）
	e.删除链接文件，不影响源文件
	f.目录可以作软链接，但需要绝对路径

2.创建软链接
	ln -s 源文件/目录 软链接名

3.作用
	a.目录迁移，保持旧路径不变，解决磁盘空间不足问题
	b.代码上线
	c.去软件版本号，优化名字长的问题
```

硬链接和软链接的区别：
```bash
1.硬链接inode号码相同，软链接inode号不同
2.目录不可作硬链接，可做软链接
3.硬链接不可跨文件系统，软链接可跨文件系统
4.目录硬链接个数默认2个，文件默认1个
5.删除硬链接不影响源文件，硬链接个数为0，则文件删除
6.删除软链接不影响源文件，删除源文件，则文件删除
7.软硬链接都是普通文件，都可使用rm删除
8.inode号都为0，并且文件没有被进程调用，文件才被真正删除
9.创建方式不同ln  和  ln -s
```
