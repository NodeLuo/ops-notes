
1、定时任务配置方式

```bash
1.修改系统配置文件
	/etc/crontab
	
	a.配置方法
		使用vim编辑/etc/crontab
	b.查看定时任务
		cat /etc/crontab
		
		[root@breadbomb ~]# cat /etc/crontab
		SHELL=/bin/bash
		PATH=/sbin:/bin:/usr/sbin:/usr/bin  （有些命令不在这里面需要自行修改）
		MAILTO=root
		
		# For details see man 4 crontabs
		
		# Example of job definition:
		# .---------------- minute (0 - 59)
		# |  .------------- hour (0 - 23)
		# |  |  .---------- day of month (1 - 31)
		# |  |  |  .------- month (1 - 12) OR jan,feb,mar,apr ...
		# |  |  |  |  .---- day of week (0 - 6) (Sunday=0 or 7) OR sun,mon,tue,wed,thu,fri,sat
		# |  |  |  |  |
		# *  *  *  *  * user-name  command to be executed


2.用户的定时任务
	/var/spool/cron/root #root定时任务配置文件
	/var/spool/cron/bob  #bob定时任务配置文件    普通用户很少用
	
	a.使用命令
		crontab -e  #相当于vim /var/spool/cron/currentUser  编辑当前用户定时任务配置文件
	b.查看定时任务
		crontab -l  #也可以cat /var/spool/cron/currentUser
	
	
	注意：crontab -e 用户的定时任务配置文件中PATH变量只有/bin/bash 和 /bin,有些命令无法匹配
	解决办法（2种）：
		（1）向crontab -e中自定义PATH变量
		（2）定时任务的命令使用绝对路径
```

2、定时任务案例

```bash
1.语法格式
	分  时  日  月  周      命令
	*   *   *   *   *   /执行脚本/命令
	
	系统定时任务需要加上user-name
	*   *   *   *   *   root   /脚本/命令
	  
	* * * * * root echo hello >> /root/a.txt
	
	（1）分 0-59
	（2）时 0-23
	（3）日 1-31
	（4）月 1-12
	（5）周 0-7（0/7都是周日）
	
	符号含义：
	（1）*     每单位
	（2）*/n   每隔n单位     * 和 */1 是一样的
	（3）a-b   区间
	（4）a,b,c 指定多个之间

2.案例 （最好都使用绝对路径，日月不要和周一起用，两条以上命令写入脚本）
	
	sh文件最好使用sh命令执行：
		（1）使用路径直接执行，需要文件属主的x权限
		（2）使用sh命令，可以直接在解释器中执行命令
	
	# 每分钟执行 
		* * * * * /root/test.sh 
	# 每天1-8点每分钟执行（到8:59）
		* 1-8 * * * /root/test.sh
	# 每天1,2点每分钟执行
		* 1,2 * * * /root/test.sh
	# 每5分钟执行 
		*/5 * * * * /root/test.sh 
	# 每小时整点执行 
		0 * * * * /root/test.sh 
	# 每天凌晨2点执行 
		0 2 * * * /root/test.sh 
	# 每天2点30分执行 
		30 2 * * * /root/test.sh 
	# 每周一凌晨3点执行 
		0 3 * * 1 /root/test.sh 
	# 每月1号早上8点执行 
		0 8 1 * * /root/test.sh 
	# 每天23点30分重启nginx 
		30 23 * * * systemctl restart nginx 
	# 每天凌晨1点备份数据库 
		0 1 * * * /usr/bin/mysqldump -uroot -p123456 dbname > /data/backup/db_$(date +\%Y\%m\%d).sql
```

3.定时任务注意事项

```bash
1.用时间命令创建文件
	date +%F    #定时任务中%无法识别，使用 \%
				#定时任务执行脚本，脚本中可以用%
	
	#创建当前日期名文件
	[root@breadbomb ~]# touch `date +%F`.txt
	[root@breadbomb ~]# ls `date +%F`.txt
	2026-05-18.txt
	
	#创建IP+时间的文件
	[root@breadbomb ~]# echo `hostname -I|sed 's# ##g'`_`date +%F`.txt
	10.0.0.200_2026-05-18.txt
	
	#创建IP+时间的文件（二）
	[root@breadbomb ~]# echo `ifconfig eth0|awk 'NR==2{print $2}'`_`date +%F`.txt
	10.0.0.200_2026-05-18.txt
	
	#创建主机名称+IP的文件
	[root@breadbomb ~]# echo `hostname`_`ifconfig eth0|awk 'NR==2{print $2}'`.txt
	breadbomb_10.0.0.200.txt
	
	#创建主机名称+IP的文件（二）
	[root@breadbomb ~]# echo `hostname`_`hostname -I|sed 's# ##g'`.txt
	breadbomb_10.0.0.200.txt
	
	#显示当前时间
	[root@breadbomb ~]# date +%F-%H:%M:%S
	2026-05-18-21:13:02

2.文件堆积问题

没有开启邮件服务的情况下：
	定时任务的结果未定向到空或者文件中，会生成大量小文件堆积到系统中，导致inode号码占满，磁盘无法写入数据
```
